Another point about Static Site Generators

Posted on July 19, 2017

Yes, static website site generators like http://gohugo.io are light on server resources and provide a great workflow. However there’s one limitation worth mentioning : editing on the go.

Indeed when using a CMS like WordPress editing options abound for the wandering blogger. WordPress offers a dedicated iOS and Android app. Moreover other dedicated editors specialize in WordPress content creation. And last, you can default to using the web interface either through wordpress.com’s slimmed-down editor or your own WordPress dashboard.

When using a static website generator, on-the-go editing is more tricky. Most workflows involve the command line and a synchronization script. There’s no such thing as a off-the-shelf mobile editor.

Web-based editors exist like https://github.com/iakta/hugocockpit but they seem to require a separate php-based installation, negating one of the main selling point of a SSG. Indeed if I have to install php packages while not stick with a WordPress of even a lighter php-based CMS. A counter argument to that is that it seems possible to run one such editor for multiple hugo instances.

There’s also an evernote-powered content editing option: [https://github.com/zzamboni/enwrite]enwrite. It requires an initial local setup but then it allows you to post content from evernote including evernote on mobile.

Something to be tested and reviewed.
